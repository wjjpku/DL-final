# 阶段性结论

## 结论 1：研究问题是成立且值得做的

proposal 的问题设置清晰，并且与最新研究趋势一致。近年的工作已经说明两件事：

- loss curve 本身存在可建模的结构，尤其在 schedule 进入模型后更明显；
- 小模型或便宜 proxy 往往能提供对大模型有用的排序信息或趋势信息。

因此，“用小模型曲线增强 schedule-aware loss prediction”不是拼接两个无关方向，而是一个自然且有潜力的结合点。

## 结论 2：最合适的研究切入点不是黑盒大模型，而是可解释 backbone + 条件化参数

直接用一个大黑盒去吞掉所有元数据和辅助曲线，虽然可能短期指标更好，但很难回答 proposal 第三个问题：辅助信号到底揭示了哪些规律，是否能导出更好的公式形式。

因此当前最稳妥的路径是：

- 先选一个 schedule-aware backbone
- 再让辅助特征只作用于少量参数或修正项

这样既能测增益，也能分析公式层面的变化。

## 结论 3：公开数据收集应该优先服务“配对样本构造”

本项目的关键样本不是单条曲线，而是：

- 一个目标大模型 run
- 一组可配对的小模型 run
- 它们共享的数据、架构族和 schedule 信息

所以数据工程重点不应放在“尽量多抓来源”，而应放在“尽量多构建高质量可配对样本”。

## 结论 4：首轮实验最值得先在 Pythia 上做

原因有三点：

- 尺度族完整，适合验证 small-to-large transfer
- 控制变量较强，便于排除数据顺序和配置差异的干扰
- 适合作为公式与特征设计的原型验证场

换言之，`Pythia` 更适合“先证伪/证实核心假设”，而 `OLMo` 与 `LLM360` 更适合“证明方法能在真实公开日志中落地”。

## 结论 5：首轮成功标准应聚焦三件事

- 是否比不使用辅助特征的 schedule-aware baseline 更准
- 是否在目标模型只观察到较短前缀时仍能提升预测
- 是否能跨 schedule 保持稳定增益

只要这三点中的前两点成立，proposal 就已经具备继续深入做理论化和公式改造的价值。

## 当前建议

### 近期

- 先建立公开来源索引表和统一 schema
- 先在 `Pythia` 构造小模型到大模型的配对关系
- 先实现一个最简单的 B2 + F3 原型

### 中期

- 扩展到 `OLMo` 与 `LLM360`
- 增加跨来源测试
- 分析 warmup 与 decay 阶段中哪些辅助统计量最稳定

### 后期

- 若观察到稳定规律，再把条件化修正项整理为新的经验公式
- 进一步讨论其与现有 temporal scaling / multi-power-law 的联系

## 一句话判断

这是一个值得立刻推进的方向，但执行上必须坚持“小而准”的原则：先用高质量配对样本验证辅助特征是否真有用，再决定是否扩展到更复杂的数据与模型。
