# Part C 审计 —— "学习率调度并非绝热"(真实约 10M 字节级 transformer)

范围:`repro/engine.py`、`repro/analyze_curves.py`、`repro/analyze_show.py`,以及展示用
结果(`results/SHOWCASE_REPORT.json`、`results/curves_show/*.csv`)。以下所有数字均通过
运行代码重新推导得出,而非仅仅阅读。每一节内部的优先级顺序为高→低。

---

## (a) 需要修复的真实 BUG

1. **[重大 —— 用错了工具来支撑核心 tau 主张 / 混叠,而不仅仅是一个 p0 bug]
   `analyze_curves.tau_vs_eta` 无法在这些调度上测量 tau,并静默返回 n=0。**
   `tau_vs_eta` 对 wsdcon drop 之后的 *MPL 残差* 拟合一个指数。但 MPL 定律
   本身已经包含一个损失-drop(LD)项(`mpl_loss` / `mpl_loss_at`),它拟合了 drop 后的
   弛豫,因此它喂给 `measure_tau` 的残差本质上是平的(幅度 ≈ 1e-9…1e-18)。
   在真实展示数据上实地验证(`wsdcon_5`,最小 eta,最瞬态):残差
   范围约为 ~0.013,但拟合找到的指数幅度约为 ~2e-18 且 r2≈0 —— 而且这一点
   **在修正 p0 猜测后仍未改变**(修正 p0 → 幅度 ~8e-18,r2≈0)。因此 n=0 的结果是
   由 **MPL-LD/弛豫混叠** 造成的,而不主要是 p0 bug 造成的。修复需要更换
   估计器,而非更换初始猜测:要么 (i) 在一个 *无 LD* 的主干上拟合弛豫(这正是
   `analyze_show.tau_backbone` 所做的),要么 (ii) 在原始损失减去一个固定幂律
   主干后测量 tau。如其所写,`tau_vs_eta` 应被视为对两阶段探针不可用。

2. **[轻微–中等 —— 真实但受限:`engine.measure_tau` 的 p0 使用样本索引单位]**
   第 266 和 279 行:tau 的 `p0` 为 `max((t1 - t0) / 5.0, 1.0)`,其中 `t1,t0` 是 **数组
   索引**。当传入显式步数轴 `t`(即它本为其设计的不规则采样情形)时,拟合 x 范围
   以步数单位计(数千),而 tau0 仍停留在 ~N_samples/5。在一个具有非平凡幅度的 *真正*
   瞬态上,这可能使优化器停滞在错误的局部极小值。
   修复:在 `t` 被切片后,基于实际拟合轴跨度来设定 tau0,例如
   `tau0 = max((t[-1] - t[0]) / 5.0, 1.0)`。对 `floor=None` 和给定 `floor` 两个
   分支(以及 `t0>0` 情形)都应用此修复。注意 —— 这个 bug 本身 **并不** 解释展示中的
   n=0:在真实步数网格上的一个合成的干净弛豫即便用有 bug 的 p0 也能被很好地恢复
   (真值 200/500/850/1500 → 199/501/855/1510,r2≥0.98),因为那里 N_samples/5 ≈ 210 恰好
   落在真值 tau 附近。该 bug 在 N_samples/5 远离真值 tau 时才会发作;在依赖任何
   显式 `t` 用法之前先修复它。样本索引调用方(NQM 探针 E1/E2/E3/G5,无显式
   `t`)不受影响。

3. **[轻微 —— lambda 网格在逐曲线拟合中被边界钉住]**
   `fit_droprelaxS` 扫描 `geomspace(0.5, 500, 40)`,`fair_baseline_test` 扫描
   `geomspace(0.3, 500, 50)`。每条曲线最优的 lambda 被钉在网格的 **下界**(核
   想要朝 lambda→0 / 累积-drop / 绝热极限方向走)。把网格向下加宽
   (例如到 1e-3)并报告 `corr(K, cumsum(drops))` 以标记退化,否则就会从一个被截断的
   扫描中得出"有限 lambda"的结论。

4. **[轻微 —— R2 被平坦的稳定阶段虚高]**
   `fit_droprelaxS` 在整条曲线上计算 R2,而其 SS_tot 的约 55% 存在于常数学习率
   稳定阶段,那里 K≈0 且 r≈0。任何单调上升的核在那里都能得高分。在
   衰减阶段的点上报告 R2,且/或对照一个稳定阶段的空模型。(wsd_sharp 上仅衰减的 R2 ≈ 0.61,而非
   0.749。)

5. **[轻微 —— 末端滞后窗口的选择粉饰了该主张]**
   `analyze_show` 的末端滞后是最后 1/7 个点的均值;wsd_sharp 的单个末步值
   为 +0.0067(对比加窗后的 +0.0104),且残差在末端是 *递减* 的 ——
   与单调衰减滞后不一致。两者都报告,或为该窗口给出理由。

数值上 CLEAN(已审计,无需修复):`droprelaxS` 递归(与暴力双重求和
匹配到 <5e-16)、`mpl_loss_at` 的 S-分箱(max_dec=600 → 损失误差约 ~1e-5,drop 质量精确守恒)、
`adamw_nqm`(真正的 AdamW —— 类符号的首步,L_eq 与 (eta/4)·Σs 匹配到 1%)。

## (b) 伪影 / 循环论证

1. **泄漏 —— `cosine` 同时在 MPL 训练集中,又被用作残差/公平基线的测试曲线。**
   `train=['constant','cosine','wsdcon_20']`。因此它的残差表现得人为地良好。
   实地验证:从公平基线测试集中剔除 cosine 会使优势从 **+0.285
   降到 +0.114**。(而且正是 cosine 使联合拟合获胜:cosine 的残差
   与 drop 核反相关,而累积-drop 对其预测错误。)只在留出
   曲线上报告。

2. **单曲线 DropRelaxS R2=0.749 与绝热基线退化。** 在 wsd_sharp 上所有
   drop 都落在约 ~0.59 的 S-跨度内,因此 `exp(-lambda·ΔS)≈1` 且 `K ≈ cumsum(drops)`
   (corr ≈ 0.9995–1.0)。单独的累积-drop 给出 R2=0.7491。有限-lambda 核在任何
   单条衰减曲线上提供 **零** 优势;速率依赖性只出现在 *联合
   跨调度* 对比中。不要把单曲线 R2 作为该核的证据来引用。

3. **wsd_sharp 的正末端滞后很大程度上是 MPL 外推误差,而非不可约滞后。** 它是一个
   留出调度;把它加入 MPL 训练集会使末端滞后塌缩约 ~5 倍(+0.0104 → ~0.002),
   而拟合目标几乎不变。在常数学习率的 *稳定* 阶段也出现大的正残差,
   那里并无衰减 —— 这对"衰减滞后"而言是不可能的。当训练集改变时残差以
   零和方式重新分配(在 wsd 上训练会把 cosine 驱动到 −0.012)。这些都是固定形式定律
   失拟的标志。然而一旦高学习率主干被锚定,符号本身是稳健的
   (见下文注意事项 1)。

4. **`tau_vs_eta` 与 `tau_backbone` 不一致,以及究竟展示的是哪一个。** 核心
   结论"tau ~ 恒定 ~850"来自 `analyze_show.tau_backbone`(其自身的 curve_fit,p0 tau=200,
   单位为正确的步数,边界 [5,8000])—— 而 **不是** 来自已审计的引擎 `measure_tau` /
   `tau_vs_eta` 路径(后者返回 n=0)。两个估计器因 (a)(1) 中的 LD-混叠而相差
   数个数量级。提示词把 tau 归因于"通过 measure_tau 的 wsdcon 两阶段探针",
   但起作用的数字来自一个独立的、手工调过的主干拟合器。披露
   这一点;不要把 `tau_vs_eta` 呈现为其来源。

5. **对平坦 tau 的截断辩护是循环的。** "对小 eta 而言真值 tau~1/eta 超过 3500 步
   的窗口"恰恰假设了正在被检验的那个标度关系。一个恢复测试(在真实网格上
   合成 tau + 真实噪声)可在窗口 *之内* 恢复直到 ~5000–8000 的 tau;一个
   在真实 wsdcon_5 上的剖面似然 sharply 拒绝 tau≫1500(tau=5000 时 SSE +288%);而且
   在 tau~850 时 `exp(-window/tau)=0.016` 意味着 >98% 的瞬态已经弛豫完成。因此平坦的
   ~850 是一个真实测量,而非一个被截断的大值。

不是伪影(经得起推敲):残差 *符号* 在 win=1(未平滑)下存活,在所有
T_MIN∈[450,1500] 下存活,在所有 MPL 种子下存活(平滑是去噪,而非捏造 —— DropRelaxS R2 随
窗口单调上升,因为它去除了不可拟合的方差)。无论 wsdcon_20 是否在 MPL 训练集中,
主干感知的 tau 都不变(<1.5%),因此它相对于定律拟合而言不是循环的。
公平基线机制(一个共享的、过原点的 kappa;累积-drop 在结构上无法
区分相同末端学习率的曲线)在机理上是正确的,且不是参数个数伪影
(即便给累积-drop 一个自由的形状指数也只把它从 0.153 提到 0.165)。

## (c) 必须出现在 Part C 文稿中的注意事项

1. **只在留出曲线上报告残差 / DropRelaxS / 公平基线指标。** cosine 在
   MPL 训练集中;诚实的公平基线优势是 **~+0.11(grad 对 sharp)**,而非 +0.285。
   +0.285 这个数字符号是真实的,但被训练集内的 cosine 曲线虚高了约 ~2.5 倍。

2. **wsd_sharp 正末端滞后的幅度依赖于拟合集(在各个有效
   训练集上 +0.0074…+0.0159;constant+wsdcon 而 *不含* cosine 甚至给出 −0.0034)。只有符号
   是稳健的,而且仅当高学习率主干被锚定时才稳健**(加入 `constant` 和一个 wsdcon 探针)。不要
   过度精确化 +0.0104 这个值。

3. **突出正滞后的无拟合佐证**(它不需要 MPL):实地验证,
   wsd_sharp 末端损失 = 1.058 > wsd_grad = 1.052(Δ ≈ +0.006,约为原始评估噪声的 ~3 倍)**尽管
   wsd_sharp 有更多的累积学习率**(S_end 12.59 对 9.22)。在任何对 S 单调的定律下,更多的
   训练 ⇒ 更低的损失;更快的 sweep 反而 *更高* 是一个真正的速率依赖滞后,
   独立于任何拟合。这是单一最强的证据,应当置于首位。

4. **不要把单曲线 DropRelaxS R2=0.749 作为有限-lambda / 速率依赖性的证据来引用。**
   它在统计上与绝热累积-drop 基线(R2=0.7491)无法区分;
   最优 lambda 被钉在网格底,且想要走得更低。明确陈述速率依赖性
   仅由联合跨调度对比(相同末端学习率,不同滞后)所证明,而
   累积-drop 无法表示这一点。

5. **陈述该展示并 *没有* 复现 tau~1/eta。** 测得的 tau 大致平坦(475…912,在 16 倍 eta
   范围上仅 1.9 倍;预测为 16 倍)。幂律斜率 p=0.176(r2=0.50),主要由
   单个 eta=8e-4 的点驱动(排除它后,p≈0.003)。该论文的残差方法
   (`tau_vs_eta`)直接返回 n=0/p=NaN。窗口截断的借口被驳倒(恢复
   测试 + 剖面似然 + >98% 弛豫完成)。把 tau~1/eta 报告为 **在此尺度上失败**,
   而非"待更长窗口而定"。

6. **披露两个估计器的不一致与混叠。** `tau_vs_eta`(论文方法)得到
   n=0,因为 MPL 的 LD 项吸收了 drop 后的弛豫;~850 这个数字来自一个
   独立的无 LD 主干拟合。在此 MPL 参数化下,两阶段 wsdcon 探针无法干净地把 MPL 绝热
   drop 响应与非绝热滞后分离开。

7. **注意信噪比。** 每步 SGD 噪声地板 + 主干波动使稳定阶段的
   max|r| 达到约 ~0.01,与所主张的 ~0.01 末端滞后相当 —— 残差特征上的信噪比
   仅约 ~3 倍。

8. **瞬态的 drop 后弛豫在原始数据中确实真实存在**(在学习率 drop 之后的
   数百步里,评估损失持续下降约 ~0.03;弛豫幅度 ~0.05 是真实的)。只有它的
   *速率依赖性*(在 16 倍 eta 上幅度和 tau 与 eta 无关)是没有支持的。

## (d) 总体裁决

Part C 真实 transformer 的证据部分支持所陈述的主张:一个非绝热滞后确实
在真实约 10M transformer 中定性地出现(稳健的正符号,并由 wsd_sharp 尽管累积学习率更多却
比 wsd_grad 收得更高这一点无拟合地佐证),但公平基线优势
因把 cosine 泄漏进 MPL 训练集而被虚高约 ~2.5 倍(诚实值 ~+0.11 而非 +0.285),单曲线
DropRelaxS 与绝热基线退化,且 tau~1/eta 不仅未被确认,反而被
反驳(测得 tau 大致平坦 ~850,p≈0.18)—— 因此只有当文稿下调那些幅度、
只报告留出指标、并把 tau~1/eta 报告为在此尺度上失败而非
"待更长窗口而定地未确认"时,该文稿才是诚实的。
