\begin{abstract}
This report studies a narrow but practically useful loss-curve prediction problem: given a
strong Multi-Power Law (MPL) baseline and only a cosine calibration curve, can we improve
prediction on warmup-stable-decay (WSD) schedules without fitting any WSD target loss?
The main difficulty is that the cosine residual is not a clean schedule-response signal.
It also contains smooth errors caused by the LR-dependent parameters of MPL, and a direct
one-dimensional projection of the residual onto an LR-drop response feature fails badly.
We propose a low-capacity correction,
\[
  \widehat L_s(t)=L_{\MPL,s}(t)+a_s\widehat\kappa_s\phi_{\lambda_s,s}(t),
\]
where the response feature is a causal LR-drop convolution, the response rate is determined
by a schedule-only Herfindahl drop concentration \(q_s\), and the locality factor \(a_s\)
is an explicit support-projection energy fraction rather than a learned gate.  The only
quantity fitted from loss residuals is one nonnegative scalar \(\widehat\kappa_s\), estimated
after projecting both the cosine residual and the response feature away from the tangent
space of MPL's LR-dependent parameters.  On the current public loss-curve repository, the
resulting \(q_2\) half-life MPL-LD response improves same-scale WSD-family MAE by
\(-29.88\%\) on average with a \(-4.67\%\) worst row and \(15/15\) wins; cross-scale
transfer gives \(-24.95\%\) mean improvement with \(30/30\) wins.  Constant and short-cosine
controls remain non-harmful.  The evidence supports the correction as a development-stage,
mechanism-motivated residual estimator, not as a universal training law.
\end{abstract}

\section{Introduction}
Learning-rate (LR) schedules strongly affect the full pretraining loss curve.  For schedule
search, it is not enough to predict the final loss after training a target schedule: the
useful task is to predict the target curve before spending that compute.  This report focuses
on the following setting:

\begin{quote}
\emph{Use a strong MPL baseline and a source cosine curve to predict WSD-family target
loss curves.  Target WSD losses are used only for evaluation.}
\end{quote}

The setting is intentionally restrictive.  MPL already models a large fraction of the
schedule dependence, so any useful correction must explain a residual that is small,
structured, and easy to overfit.  Moreover, cosine is a smooth schedule.  A residual pattern
learned on cosine can mix together two different effects: a transferable LR-drop response and
a non-transferable smooth drift caused by MPL parameter error.  This is why a high-capacity
residual basis, a family-specific route, or a learned gate can look attractive numerically but
is not a satisfactory explanation.

The current contribution is a compact residual model with three design constraints.
\begin{itemize}\itemsep2pt
  \item The shape of the correction is computed only from the target LR schedule.
  \item The amplitude is fitted only from the source cosine residual.
  \item The nuisance removal has an MPL-specific interpretation: it removes local errors in
  MPL's LR-dependent tangent directions, rather than an arbitrary low-frequency basis.
\end{itemize}

The result is a single-formula correction,
\begin{equation}\label{eq:main_law}
  \widehat L_s(t)=L_{\MPL,s}(t)+a_s\widehat\kappa_s\phi_{\lambda_s,s}(t).
\end{equation}
Here \(L_{\MPL,s}\) is the frozen MPL prediction; \(\phi_{\lambda_s,s}\) is a causal
LR-drop response; \(\lambda_s\) is chosen from the effective concentration of the target
LR drops; \(a_s\) removes diffuse full-horizon schedule modes; and \(\widehat\kappa_s\) is
the only residual-fitted scalar.

\paragraph{Main findings.}
The current formula gives the following picture on the repository used in this project.
First, raw cosine residual fitting is unsafe: without MPL-LD nuisance removal, the WSD-family
MAE worsens by \(+602.17\%\) on average and wins \(0/15\) rows.  Second, MPL-LD tangent
projection recovers a stable transferable signal.  Third, replacing the older max-drop
response rule by the Herfindahl concentration \(q_2\) and interpreting the locality factor as
a support projection gives essentially the same numerical performance, but with a cleaner
story and no extra fitted parameters.  The current version improves same-scale WSD-family
MAE by \(-29.88\%\) and cross-scale WSD-family MAE by \(-24.95\%\), with all WSD rows
improving and all controls non-harmful.

\section{Problem Setup}
\paragraph{Baseline.}
We use the published MPL form as a frozen baseline.  In simplified notation,
\begin{equation}\label{eq:mpl}
L_{\MPL}(t)
=L_0+A S(t)^{-\alpha}
+B\sum_{k\le t}\Delta\eta_k\,
G\!\left(\eta_k^{-\gamma}(S(t)-S(k))\right),
\qquad
S(t)=\sum_{i\le t}\eta_i .
\end{equation}
The LR-dependent MPL parameters are \((B,C,\beta,\gamma)\), where \(G\) is the saturating
kernel used by MPL.  We do not refit MPL in this report; it is treated as the base predictor
whose residual we try to explain.

\paragraph{Data and evaluation.}
The source calibration curve is \code{cosine\_72000.csv}.  The WSD-family targets are five
curves at each of three scales \(25\)M, \(100\)M, and \(400\)M:
\code{wsd\_20000\_24000}, \code{wsdld\_20000\_24000}, and three \code{wsdcon} endpoints.
Controls are \code{cosine\_24000}, \code{constant\_24000}, and \code{constant\_72000}.
Same-scale evaluation gives \(3\times5=15\) WSD rows; cross-scale evaluation gives
\(3\times2\times5=30\) WSD rows.  We report percentage MAE change relative to MPL:
\begin{equation}
  \Delta_{\mathrm{MAE}} =
  100\cdot
  \frac{\mathrm{MAE}(\widehat L)-\mathrm{MAE}(L_{\MPL})}
       {\mathrm{MAE}(L_{\MPL})}.
\end{equation}
Negative values are improvements.

\paragraph{Calibration rule.}
The residual coefficient is fitted on the suffix \(t\ge 8000\) of the source cosine curve.
This gives \(N_{\mathrm{cal}}=500\) logged calibration points, so the ridge in the one-dimensional
projection is \(1/N_{\cal}=0.002\).  The target losses are not used to fit
\(\lambda_s\), \(a_s\), \(\widehat\kappa_s\), or any other quantity.

\section{Method}
\subsection{Causal LR-Drop Response}
Let the positive LR drop be
\begin{equation}\label{eq:drop}
  d_t=[\eta_{t-1}-\eta_t]_+ .
\end{equation}
The response feature is a causal one-pole recursion in cumulative-LR time:
\begin{equation}\label{eq:feature_recursion}
  z_t=\exp(-\lambda_s\eta_t)z_{t-1}+d_t,\qquad
  \phi_{\lambda_s,s}(t)=\frac{z_t}{\eta_{\max}},\qquad z_0=0 .
\end{equation}
Equivalently,
\begin{equation}\label{eq:feature_sum}
  \phi_{\lambda_s,s}(t)
  =
  \frac{1}{\eta_{\max}}
  \sum_{u\le t} d_u
  \exp\!\left(-\lambda_s\sum_{v=u+1}^{t}\eta_v\right).
\end{equation}
This feature has three important properties.  It is causal, because only past LR drops
contribute.  It is drop-only, because constant LR intervals do not inject new response.  It
has a single response channel, so it does not use schedule-family labels or hand-routed
cases.

\subsection[q2 Half-Life Response Rate]{\(q_2\) Half-Life Response Rate}
The response rate \(\lambda_s\) should be schedule-dependent, but not target-loss-dependent.
We first convert the positive LR drops into a probability distribution:
\begin{equation}\label{eq:q2_def}
  D_s=\sum_t d_t,\qquad
  p_t=\frac{d_t}{D_s},\qquad
  q_s=\sum_t p_t^2=\frac{1}{n_{\mathrm{eff}}}.
\end{equation}
When the LR drop happens in one step, \(q_s=1\).  When the same total drop is spread
uniformly over \(n\) steps, \(q_s=1/n\).  Thus \(q_s\) is an effective drop concentration,
not a learned label.

The loss curves are logged mainly every \(\Delta_{\obs}=128\) training steps.  We therefore
anchor the response rate to the observable half-life scale
\begin{equation}\label{eq:lambda_obs}
  \lambda_{\obs}
  =
  \frac{\log 2}{\eta_{\max}\Delta_{\obs}} .
\end{equation}
The current rule linearly interpolates the response half-life between two observation
intervals for diffuse drops and one observation interval for a single sharp drop:
\begin{equation}\label{eq:q2_halflife}
  H_s=(2-q_s)\Delta_{\obs},\qquad
  \lambda_s=\frac{\log 2}{\eta_{\max}H_s}
  =\frac{\lambda_{\obs}}{2-q_s}.
\end{equation}
This is deliberately modest.  It does not claim a universal physical relaxation constant; it
only says that a sharper LR drop should be resolved on a shorter observed response time than
a diffuse LR decay.  In the current data, the rule gives \(\lambda_s\approx 9.03\) for WSD
sharp/linear and \(\lambda_s=18.05\) for the single-step WSD-con targets.

\subsection{Support-Projection Locality}
Without a locality boundary, a local WSD cooldown correction can be transferred onto a
full-horizon diffuse cosine decay.  The current formula uses
\begin{equation}\label{eq:locality}
  a_s
  =
  \mathbf 1\{D_s>0\}
  \frac{\|(I-P_u)m_s\|_2^2}{\|m_s\|_2^2}
  =
  \mathbf 1\{D_s>0\}
  \left[1-\frac{\ell_s}{T_s-W}\right]_+ .
\end{equation}
Here \(T_s-W\) is the post-warmup horizon, \(\ell_s\) is the span of the positive-drop
support, \(m_s\) is the uniform density on that support, and \(u\) is the uniform diffuse
mode on the full post-warmup horizon.  The equality follows from
\(\|m_s\|_2^2=1/\ell_s\) and \(\|P_u m_s\|_2^2=1/(T_s-W)\).  Therefore \(a_s\) is not a
learned gate: it is the remaining local forcing energy after removing the full-horizon
diffuse mode.  It is \(0.8168\) for finite WSD sharp/linear cooldowns, approximately \(1\)
for WSD-con single drops, and \(0\) for the short-cosine and constant controls.

\subsection{MPL-LD Tangent Nuisance Projection}
The central identification issue is that the source cosine residual is contaminated by MPL
parameter error.  Write
\begin{equation}
  r_{\cosine}(t)=L_{\cosine}(t)-L_{\MPL,\cosine}(t).
\end{equation}
If the LR-dependent MPL parameters are locally misestimated, then a first-order residual
component has the form
\begin{equation}
  J_{\LD}(t)\Delta\theta_{\LD},\qquad
  J_{\LD}(t)=
  \left[
  \frac{\partial L_{\MPL}}{\partial\log B},
  \frac{\partial L_{\MPL}}{\partial\log C},
  \frac{\partial L_{\MPL}}{\partial\log\beta},
  \frac{\partial L_{\MPL}}{\partial\log\gamma}
  \right].
\end{equation}
We compute this tangent basis by finite differences of the frozen MPL formula on the source
cosine suffix.  Let \(P_{\LD}\) be the orthogonal projection onto the columns of \(J_{\LD}\).
For a target schedule \(s\), define
\begin{equation}\label{eq:projection_xy}
  x_s=(I-P_{\LD})\phi_{\lambda_s,\cosine},\qquad
  y=(I-P_{\LD})r_{\cosine}.
\end{equation}
The amplitude is the nonnegative ridge projection
\begin{equation}\label{eq:kappa}
  \widehat\kappa_s
  =
  \frac{\langle x_s,y\rangle_+}
       {\|x_s\|_2^2+1/N_{\mathrm{cal}}} .
\end{equation}
The projection is target-aware only through the schedule-derived \(\lambda_s\).  It never
uses target loss values.  The nonnegative truncation encodes the observed sign: the residual
we want to transfer is a positive loss excess after LR drops.

\subsection{Training and Testing Procedure}
The calibration procedure is:
\begin{enumerate}\itemsep2pt
  \item Load the source \code{cosine\_72000.csv} curve for a source scale.
  \item Compute the frozen MPL prediction and source residual \(r_{\cosine}\).
  \item For each target schedule, compute \(d_t\), \(q_s\), \(\lambda_s\), and \(a_s\) from
  the LR schedule only.
  \item Build the source response feature \(\phi_{\lambda_s,\cosine}\), project it and the
  source residual away from MPL-LD tangent directions, and compute \(\widehat\kappa_s\).
\end{enumerate}
The prediction procedure is:
\begin{enumerate}\itemsep2pt
  \item Build the target response feature \(\phi_{\lambda_s,s}\) from the target LR schedule.
  \item Output Eq.~\eqref{eq:main_law}.
  \item Use the target loss curve only to compute evaluation MAE.
\end{enumerate}

\section{Why the Formula is Plausible}
\paragraph{Linear response motivation.}
The response feature follows from a first-order tracking-error approximation.  Let an
unobserved excess-loss state \(e_t\) track an LR-dependent equilibrium.  Around the current
trajectory, a positive LR decrement perturbs the state by a forcing proportional to \(d_t\),
and the perturbation decays under the local optimizer dynamics:
\begin{equation}
  e_t \approx \rho_t e_{t-1}+b d_t,\qquad
  \rho_t \approx \exp(-\lambda\eta_t).
\end{equation}
Projecting this state onto loss gives Eq.~\eqref{eq:feature_recursion} up to a scalar
amplitude.  This derivation is only a local approximation, but it explains why the feature is
causal, why constant LR creates no new response, and why the memory should be measured in
cumulative LR rather than raw step count.

\paragraph{Why not fit the response directly?}
Directly fitting \(\kappa\) from cosine residuals assumes
\(r_{\cosine}\approx\kappa\phi_{\lambda,\cosine}\).  The audit rejects this assumption:
the no-nuisance raw projection worsens WSD-family MAE by hundreds of percent.  The reason is
not that the response feature is useless, but that cosine residuals contain a smooth component
aligned with MPL's LR-dependent parameter errors.  Eq.~\eqref{eq:projection_xy} makes the
identification statement explicit:
\begin{equation}
  r_{\cosine}
  =
  \kappa\phi_{\lambda,\cosine}
  +J_{\LD}\Delta\theta_{\LD}
  +\epsilon .
\end{equation}
After applying \(I-P_{\LD}\), the nuisance term is removed to first order, leaving a
one-dimensional estimate of the transferable response amplitude.

\paragraph{Why \(q_2\), not a family label?}
The model needs to distinguish a single-step drop from a smooth distributed decay.  A family
label would solve this numerically but would not generalize.  The concentration
\(q_s=\sum_t p_t^2\) is a schedule statistic with a direct meaning: it is the inverse
effective number of drop atoms.  This is more interpretable than the older max-drop ratio
\(q_\infty=\max_t p_t\), while producing nearly identical results in the current audit.

\paragraph{Why support projection, not a gate?}
The locality factor is often tempting to describe as a gate, but that is misleading.  It is
the exact energy fraction left when a uniform forcing on the LR-drop support is projected
away from the uniform full-horizon mode.  This matters because full-horizon cosine decay can
have many small positive drops, but it should not receive a local cooldown transient
correction.

\section{Discussion}
\paragraph{What the current result supports.}
The strongest supported claim is not that we have discovered a universal optimizer law.  The
supported claim is more specific: after removing local MPL-LD tangent drift from the cosine
residual, a one-dimensional causal LR-drop response transfers reliably from cosine to the
current WSD-family targets.  The response amplitude is low capacity, the response shape is
schedule-only, and the main evaluation uses no target-loss fitting.

\paragraph{Why the model is not just overfitting.}
The formula has only one residual-fitted scalar for a source scale and response operator.
The response rate, locality factor, and nuisance subspace are computed from the LR schedule
and the frozen MPL formula.  Several higher-capacity alternatives were intentionally rejected:
DCT projection is numerically strong but less interpretable and less stable cross-scale;
density projection is natural but harms controls; no-locality improves WSD means but damages
the short-cosine control; raw projection fails completely.  These negative results are part
of the model selection evidence.

\paragraph{Remaining limitations.}
The current result is still a development candidate.  The \(q_2\) half-life bracket uses the
logging interval as a structural prior; it is cleaner than the older endpoint constants, but
it is not a theorem about training dynamics.  The suffix start \(8000\) is a protocol choice
that removes early cosine transients; later starts remain positive but reduce the gain.  The
support-projection locality explains the safety boundary for current controls, but it is not
a complete dynamical derivation of all schedule families.  Finally, all main numbers come
from the current loss-curve repository; external schedules and new training runs are needed
before making broader claims.

\section{Conclusion}
This paper rewrites the current project around one core line: MPL is the baseline, cosine
residuals are contaminated by MPL LR-dependent drift, and a causal LR-drop response becomes
useful only after MPL-LD tangent nuisance projection.  The final working formula is
\[
  \widehat L_s(t)=L_{\MPL,s}(t)+a_s\widehat\kappa_s\phi_{\lambda_s,s}(t),
  \qquad
  \lambda_s=\frac{\lambda_{\obs}}{2-q_s},
  \qquad
  q_s=\sum_t\left(\frac{d_t}{\sum_u d_u}\right)^2 .
\]
With support-projection locality, the method improves all current WSD-family rows
(\(-29.88\%\) same-scale mean, \(-24.95\%\) cross-scale mean) while leaving controls
non-harmful.  The main value of the result is interpretability under a strict cosine-only
calibration protocol: it keeps the correction small, schedule-derived, and explicit about
what has not yet been externally validated.

\section{Implementation Details}
\paragraph{Finite-difference tangent basis.}
The MPL-LD tangent basis is computed by perturbing the four LR-dependent MPL directions
\[
  \log B,\qquad \log C,\qquad \log\beta,\qquad \log\gamma
\]
around the frozen MPL parameters, then evaluating the resulting loss-curve differences on
the source cosine suffix.  The projection is an ordinary least-squares orthogonal projection
onto these four columns.

\paragraph{Cross-scale evaluation.}
For same-scale rows, the source and target scale are identical.  For cross-scale rows, a
source cosine residual from one scale is used to calibrate \(\widehat\kappa_s\), and the
result is applied to a target curve at another scale.  There are \(3\times2\times5=30\)
cross-scale WSD rows.  The current formula improves all of them.
